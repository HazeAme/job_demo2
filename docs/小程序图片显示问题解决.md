# 小程序图片显示问题解决

## 问题1：图片渲染不出来

### 原因
后端返回的图片URL是相对路径（如 `/uploads/xxx.jpg`），小程序需要完整的URL才能显示。

### 解决方案
已在 `history.js` 中修复，自动将相对路径拼接为完整URL：

```javascript
// 处理图片URL，将相对路径转为完整URL
item.images = item.images.map(img => {
  if (img && img.startsWith('/uploads/')) {
    return app.globalData.baseUrl + img
  }
  return img
})
```

## 问题2：需要配置 downloadFile 域名

在小程序后台配置图片下载域名：

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 「开发」→「开发管理」→「开发设置」→「服务器域名」
3. 在 **downloadFile合法域名** 中添加：
   ```
   http://localhost:8080
   ```
   或你的实际域名：
   ```
   https://your-domain.com
   ```

## 问题3：开发者工具设置

在开发阶段，可以跳过域名校验：

1. 微信开发者工具 → 右上角「详情」
2. 「本地设置」→ 勾选「不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书」

## 完整的服务器域名配置

| 域名类型 | 需要配置的域名 |
|---------|---------------|
| request合法域名 | `http://localhost:8080` 或你的后端域名 |
| downloadFile合法域名 | `http://localhost:8080` 或你的后端域名 |
| uploadFile合法域名 | `http://localhost:8080` 或你的后端域名 |

## 测试图片显示

1. 确保后端服务已启动
2. 确保已上传至少一张图片的打卡记录
3. 打开历史页面
4. 检查图片是否正常显示

## 调试方法

在开发者工具控制台查看：
```javascript
// 查看图片URL是否正确
console.log(item.images)

// 应该输出完整的URL：
// ["http://localhost:8080/uploads/xxx.jpg"]
```
