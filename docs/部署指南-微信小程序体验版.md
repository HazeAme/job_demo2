# 微信小程序体验版部署指南

## 一、准备工作

### 1. 注册微信小程序账号

1. 访问 [微信公众平台](https://mp.weixin.qq.com/)
2. 点击右上角「立即注册」
3. 选择「小程序」类型
4. 填写邮箱、密码等信息完成注册
5. 进行主体认证（个人开发者选择个人主体）

### 2. 获取 AppID

注册完成后，在小程序后台：
- 左侧菜单「开发」→「开发管理」→「开发设置」
- 找到「AppID(小程序ID)」，复制备用

### 3. 下载微信开发者工具

- 访问 [微信开发者工具下载页](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
- 下载并安装对应系统版本

---

## 二、配置小程序

### 1. 修改项目配置

打开 `miniprogram/project.config.json`，确保有正确的 appid：

```json
{
  "appid": "你的小程序AppID"
}
```

### 2. 配置服务器域名

在小程序后台「开发」→「开发管理」→「开发设置」→「服务器域名」中配置：

**request合法域名：**
- 如果使用内网穿透：`https://你的cpolar域名.cpolar.cn`
- 如果部署到云服务器：`https://你的服务器域名`

**uploadFile合法域名：**（图片上传需要）
- 同上

**downloadFile合法域名：**（图片下载需要）
- 同上

**注意：** 域名必须备案，且只支持 HTTPS

### 3. 修改小程序后端地址

打开 `miniprogram/app.js`，修改 baseUrl：

```javascript
App({
  globalData: {
    baseUrl: 'https://你的后端域名'  // 改为实际可访问的地址
  }
})
```

---

## 三、上传代码

### 1. 导入项目

1. 打开微信开发者工具
2. 点击「+」或「导入项目」
3. 选择 `miniprogram` 文件夹
4. 填入你的 AppID
5. 点击「确定」

### 2. 测试功能

在开发者工具中：
- 点击「编译」查看效果
- 测试登录、打卡、上传图片等功能
- 确保所有功能正常

### 3. 上传代码

1. 点击右上角「上传」按钮
2. 填写版本号（如：1.0.0）
3. 填写项目备注（如：面试演示版本）
4. 点击「上传」

---

## 四、设置为体验版

### 1. 登录小程序后台

访问 [微信公众平台](https://mp.weixin.qq.com/)，扫码登录

### 2. 提交体验版

1. 左侧菜单「管理」→「版本管理」
2. 在「开发版本」中找到刚上传的版本
3. 点击「提交审核」旁边的「体验版」按钮
4. 点击「选为体验版」

### 3. 获取体验二维码

1. 体验版设置成功后，点击「体验版」按钮
2. 会显示体验二维码
3. 你可以：
   - 下载二维码图片发给HR
   - 让HR直接扫码体验

---

## 五、HR体验流程

### 方式一：扫码体验（推荐）

1. 将体验二维码截图或保存
2. 发送给HR
3. HR使用微信扫码即可进入小程序

### 方式二：添加体验成员

如果HR需要多次访问：

1. 小程序后台「管理」→「成员管理」
2. 点击「添加成员」
3. 输入HR的微信号
4. 选择「体验成员」权限
5. HR在微信中搜索「小程序助手」即可找到你的小程序

## 六、后端部署选项

### 选项A：内网穿透（快速演示）

使用 cpolar 将本地后端暴露到公网：

```bash
# 安装 cpolar
npm install -g cpolar

# 启动穿透（后端端口8080）
cpolar http 8080

# 获得 https 链接，如：https://xxxx.cpolar.cn
```

**注意：** 免费版域名会变化，每次重启需要重新配置小程序服务器域名

### 选项B：部署到云服务器（推荐）

1. 购买阿里云/腾讯云轻量服务器（约50元/月）
2. 部署后端服务
3. 配置域名和HTTPS
4. 小程序后台配置服务器域名

## 七、常见问题

### 1. 提示 "不在以下 request 合法域名列表中"

**解决：** 在小程序后台「服务器域名」中添加你的后端域名，然后：
- 开发者工具：点击「详情」→「本地设置」→勾选「不校验合法域名...」（仅开发时）
- 或等待10分钟后重启开发者工具

### 2. 图片上传失败

**解决：** 
- 确保 uploadFile 合法域名已配置
- 后端允许跨域上传

### 3. 地图定位失败

**解决：**
- 在小程序后台「开发」→「开发管理」→「接口设置」中申请「获取地理位置」权限
- 在 app.json 中已配置 `requiredPrivateInfos`

### 4. 体验版无法访问

**检查：**
- 后端服务是否正常运行
- 服务器域名是否正确配置
- 网络是否通畅
