# 小程序位置模块重构设计文档

## 日期
2025-01-14

## 背景
当前位置功能使用高德Web API + 后端代理方案，但存在Key类型不匹配问题。现改用腾讯位置服务SDK，实现内嵌实时小地图。

## 需求

### 功能需求
1. **GPS定位** - 点击按钮获取当前位置
2. **逆地理编码** - 自动获取最近的商铺/地点名称
3. **内嵌小地图** - 实时显示当前位置，可缩放拖动
4. **搜索定位** - 搜索地点名称，手动选择位置

### 非功能需求
- 地图内嵌在打卡页面
- 流畅的交互体验
- 错误处理和降级方案

## 技术方案

### 地图服务选择
**腾讯位置服务（微信小程序SDK）**

理由：
- 微信生态原生支持
- 功能完整（定位、逆地理、搜索、地图）
- 免费额度充足（日1万次）
- 小程序直接调用，无需后端代理

### 组件设计

#### 1. 地图组件 (map)
```xml
<map
  id="locationMap"
  longitude="{{location.longitude}}"
  latitude="{{location.latitude}}"
  scale="16"
  show-location
  markers="{{markers}}"
  bindmarkertap="onMarkerTap"
  bindregionchange="onRegionChange"
  style="width: 100%; height: 300rpx;"
/>
```

#### 2. 地图控制
- **自动定位**：获取GPS后，地图中心移动到当前位置
- **手动选点**：拖动地图或点击地图空白处，更新选中位置
- **标记点**：显示当前选中位置的标记

#### 3. 位置获取流程
```
用户点击"获取当前位置"
    ↓
wx.getLocation (获取GPS坐标)
    ↓
qqmapsdk.reverseGeocoder (逆地理编码)
    ↓
获取最近POI名称
    ↓
更新地图中心 + 显示标记
    ↓
显示位置名称
```

#### 4. 搜索流程
```
用户输入关键词
    ↓
qqmapsdk.search (地点搜索)
    ↓
显示搜索结果列表
    ↓
用户选择地点
    ↓
地图中心移动到该地点
    ↓
更新位置信息
```

### API设计

#### 腾讯SDK初始化
```javascript
const QQMapWX = require('../../libs/qqmap-wx-jssdk.min.js')
const qqmapsdk = new QQMapWX({
  key: 'IPVBZ-PC46Z-BWFXB-T2RJ3-ADPZZ-EOBWC'
})
```

#### 核心方法
| 方法 | 功能 | 腾讯SDK API |
|-----|------|------------|
| `getCurrentLocation()` | 获取当前位置 | `wx.getLocation` + `reverseGeocoder` |
| `searchLocation(keyword)` | 搜索地点 | `qqmapsdk.search` |
| `selectLocation(lat, lng, name)` | 选择位置 | 更新地图中心 |
| `onMapTap(e)` | 点击地图 | 获取点击坐标，逆地理编码 |

### 交互细节
1. **地图拖动**：实时更新中心点坐标
2. **拖动结束**：触发逆地理编码，获取地点名称
3. **点击标记**：显示地点信息气泡
4. **搜索选择**：地图飞动画到选中位置

## 错误处理

| 错误场景 | 处理方案 |
|---------|---------|
| 定位权限被拒绝 | 提示用户去设置开启权限 |
| 腾讯API调用失败 | 降级到微信原生地图选点 |
| 搜索无结果 | 提示"未找到相关地点" |
| 网络错误 | 提示"网络异常，请重试" |

## 配置要求

### 小程序后台配置
需要在微信公众平台配置：
- `request合法域名`: `https://apis.map.qq.com`

### 需要下载的SDK
- 腾讯地图微信小程序SDK: `qqmap-wx-jssdk.min.js`

## 风险与应对

| 风险 | 应对 |
|-----|------|
| 腾讯Key未开启WebService | 用户需要去控制台开启 |
| 地图组件性能问题 | 控制地图高度，懒加载 |
| 坐标偏移问题 | 使用gcj02坐标系 |
